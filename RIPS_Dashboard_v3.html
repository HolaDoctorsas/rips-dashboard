🔐 Cómo funciona: el dashboard inicia sesión con tu cuenta Microsoft, lee el Excel de OneDrive directamente y se actualiza automáticamente cada vez que abres la página. Necesitas registrar una App en Azure AD una sola vez.
Azure Portal → App registrations → tu app → Application (client) ID
Azure Portal → App registrations → tu app → Directory (tenant) ID

📋 Guía rápida: cómo registrar la App en Azure (clic para expandir)
1. Ve a portal.azure.com → Microsoft Entra ID → App registrations → New registration
2. Nombre: RIPS Dashboard · Supported account types: Single tenant
3. Redirect URI: Single-page application (SPA) → escribe http://localhost (o la URL donde alojes el HTML)
4. Clic en Register. Copia el Application ID y el Tenant ID en los campos de arriba.
5. Ve a API permissions → Add a permission → Microsoft Graph → Delegated → agrega: Files.Read y User.Read
6. Clic en Grant admin consent (necesitas ser admin del tenant, o pedírselo a TI).

⚠️ El problema: Abriste el archivo con doble clic (file://). Chrome y Edge bloquean las conexiones a internet desde archivos locales, por eso MSAL no puede cargar.
✅ Solución: 3 opciones (elige la más fácil)
🥇 Opción 1 — Python (más fácil, 1 comando)
Abre una terminal en la carpeta del archivo y ejecuta:
python -m http.server 8080
Luego abre en Chrome: http://localhost:8080/RIPS_Dashboard_v2.html
🥈 Opción 2 — Node.js
npx serve .
Abre la URL que aparezca en la terminal + /RIPS_Dashboard_v2.html
🥉 Opción 3 — VS Code Live Server
Instala la extensión Live Server en VS Code → clic derecho en el archivo HTML → Open with Live Server
💡 También debes agregar la URL del servidor en Azure AD → Autenticación → URIs de redirección SPA.
Ejemplo: http://localhost:8080
